# Deploying a client + server + database application on the internet with Render

The goal of this lecture is to display the same information as we did on lecture 7 on a public web site.

There are several cloud services that can be used to create three-tier web applications (frontend, backend and database).
This lecture, we will look at [Render](https://render.com/), which has a generous (but a little limited) free tier.
Previous years, we have used [Heroku](https://www.heroku.com/), but they have gradually made their free tier more
unusable. As American technology companies are becoming increasingly hostile to Norwegian and European interests, all
options are either limited, expensive or risky. Students are encouraged to look for other hosting alternatives in the
future.

## Set up the repository as a service on Render

1. Sign up to Render using your GitHub Account
2. Create your single free project and add a web service to it
3. For the web service, select your GitHub project and set up Language, Branch, Region, Build Command and Start Command
   1. Language: `Node`
   2. Branch: Normally `main` (for me, `lecture/08`)
   3. Region: Choose the closest geographically
   4. Build command: `npm run build`
   5. Start command: `npm start`
   6. Instance Type: `Free`
4. Deploy web service

The web service will fail to build and run, because we have to set up the correct scripts locally and push to GitHub

## Create the scripts

1. `npm pkg set scripts.build="vite build"`
2. `npm pkg set scripts.postinstall="cd server && npm install"`
3. `npm pkg set scripts.start="cd server && npm start"`
4. `cd server`
5. `npm pkg set scripts.start="tsx server.ts"`
6. Add `dist` to `.gitignore` (will be generated by `npm build`)
7. Commit and push
8. In the render dashboard for the Web service, select Manual Deploy > Latest to build the web service again

Render will automatically detect that your repository has changed and restart the build. You can now do click the automatically generated URL in Render to see your website, but it still doesn't work so well.

## Serve the react application with Hono

When you visit the website, you will receive "404 Not found". This is because even though you told Render to use `vite` to build your React code, you didn't Hono to display it. This can be fixed with a single line in `server/server.ts`

```ts
// at the start of the file
import { serveStatic } from "@hono/node-server/serve-static";

// at the end of the file
app.use("*", serveStatic({ root: "../dist" }));
```

(You can test this by running `npm start` and visiting the application at http://localhost:3000)

## Set up the database

You should now see regional borders on your map, but not the schools. This is because we have not set up the database with Render

1. In Render, in the top right corner, select `+ New` > `Postgres`. Add a name and select the free plan
2. Navigate back to your Web Service, Select `Environment`. Under `Environment` > `Environment variables` > `+ Add` select your database. This will add the environment variable `DATABASE_URL` to `process.env` in Hono. Press `Save, Rebuild and Deploy`
3. You have to update `server/server.ts` to use this environment variable. Change the line starting with `const postgres = new pg.Pool` to the following
   ```ts
   const postgres = new pg.Pool({
     connectionString:
       process.env.DATABASE_URL || "postgresql://postgres:@localhost",
   });
   ```
4. Make a script to load the data on Render:
   - On Windows: `npm pkg set scripts.db:load:server="echo create extension if not exists postgis | psql $DATABASE_URL && npm run db:grunnskoler:download && psql $DATABASE_URL < tmp/Befolkning_0000_Norge_25833_Grunnskoler_PostGIS.sql && npm run db:fylker2023:download && psql $DATABASE_URL < tmp/Basisdata_0000_Norge_25833_Fylker_PostGIS.sql"`
   - On Linux/Mac: `npm pkg set scripts.db:load:server='echo create extension if not exists postgis | psql $DATABASE_URL && npm run db:grunnskoler:download && psql $DATABASE_URL < tmp/Befolkning_0000_Norge_25833_Grunnskoler_PostGIS.sql && npm run db:fylker2023:download && psql $DATABASE_URL < tmp/Basisdata_0000_Norge_25833_Fylker_PostGIS.sql'`
5. Push the code to GitHub
6. In Render, under your Web Service, select Settings > Build & Deploy and change the Build command to `npm install && npm run build && npm run db:load:server`
7. This will download and load the data from GeoNorge.no. After it has been run successfully, you should change the Build command back to `npm install && npm run build` to avoid duplicate data in the database

Here is what the `scripts` section of your [`package.json`](./package.json) should look like:

```json
{
  "test": "tsc --noEmit && prettier --check .",
  "prepare": "husky",
  "dev": "vite",
  "psql": "docker exec -i postgis /usr/bin/psql --user postgres",
  "db:load:server": "echo create extension if not exists postgis | psql $DATABASE_URL && npm run db:grunnskoler:download && psql $DATABASE_URL < tmp/Befolkning_0000_Norge_25833_Grunnskoler_PostGIS.sql && npm run db:fylker2023:download && psql $DATABASE_URL < tmp/Basisdata_0000_Norge_25833_Fylker_PostGIS.sql",
  "db:grunnskoler:download": "download --out tmp/ --extract https://nedlasting.geonorge.no/geonorge/Befolkning/Grunnskoler/PostGIS/Befolkning_0000_Norge_25833_Grunnskoler_PostGIS.zip",
  "db:grunnskoler:import": "npm run psql < tmp/Befolkning_0000_Norge_25833_Grunnskoler_PostGIS.sql",
  "db:fylker2023:download": "download --extract --out tmp/ https://nedlasting.geonorge.no/geonorge/Basisdata/Fylker2023/PostGIS/Basisdata_0000_Norge_25833_Fylker2023_PostGIS.zip",
  "db:fylker2023:import": "npm run psql < tmp/Basisdata_0000_Norge_25833_Fylker_PostGIS.sql",
  "build": "vite build",
  "postinstall": "cd server && npm install",
  "start": "cd server && npm start"
}
```

You should now be able to see your data on your deployed Render website.

Notice that you can view the database service on Render and find information under Info > General on how to connect to the database from IntelliJ
